PROJECT OVERVIEW
────────────────────────────────────────────────────────────────────────────────
An Explainable AI Framework for Climate-Responsive Building Design and 
Sustainable Material Selection in Tamil Nadu, inspired by Laurie Baker Principles.

Tech Stack:
- Backend: Python 3.10+
- Frontend: Streamlit (Web UI)
- Visualization: Matplotlib
- Data: Pandas, NumPy

═══════════════════════════════════════════════════════════════════════════════
FRONTEND DEPLOYMENT OPTIONS
═══════════════════════════════════════════════════════════════════════════════

OPTION 1: Streamlit Cloud (Easiest)
────────────────────────────────────────────────────────────────────────────────
Platform:   Streamlit Cloud (free tier available)
Setup:      Connect GitHub repo → Deploy
Pros:       Zero configuration, automatic updates, free SSL
Cons:       Limited resources, Streamlit-only
URL:        https://share.streamlit.io/


OPTION 2: Heroku (Simple)
────────────────────────────────────────────────────────────────────────────────
Platform:   Heroku
Setup:      Procfile + requirements.txt
Pros:       Easy deployment, add-ons available
Cons:       Paid plans required for production
Command:    heroku create && git push heroku main


OPTION 3: Docker + Cloud Run (Scalable)
────────────────────────────────────────────────────────────────────────────────
Platform:   Google Cloud Run / AWS ECS / Azure Container Instances
Setup:      Dockerfile → Container Registry → Deploy
Pros:       Scalable, pay-per-use, full control
Cons:       More complex setup
Example:    gcloud run deploy --image gcr.io/project/app


OPTION 4: VPS (Full Control)
────────────────────────────────────────────────────────────────────────────────
Platform:   DigitalOcean / Linode / AWS EC2
Setup:      Nginx + Gunicorn + Streamlit
Pros:       Full control, custom domain, SSH access
Cons:       Manual server management
Stack:      Ubuntu + Nginx + Supervisor + Streamlit


OPTION 5: Serverless (Cost-effective)
────────────────────────────────────────────────────────────────────────────────
Platform:   AWS Lambda + API Gateway
Setup:      Zappa / Serverless Framework
Pros:       Pay-per-request, auto-scaling
Cons:       Cold start latency, limited execution time
Note:       Better for API backend, not ideal for Streamlit

FRONTEND PERFORMANCE OPTIMIZATION

1. Caching Strategy
────────────────────────────────────────────────────────────────────────────────
@st.cache_data                          # Cache data loading
@st.cache_resource                      # Cache expensive objects
st.experimental_memo                    # Memoize function results
st.experimental_singleton               # Singleton pattern

Example:
  @st.cache_data(ttl=3600)
  def load_climate_data():
      return pd.read_csv("climate.csv")


2. Lazy Loading
────────────────────────────────────────────────────────────────────────────────
- Load heavy modules only when needed
- Defer agent initialization
- Progressive image loading
- Pagination for large datasets


3. Code Splitting
────────────────────────────────────────────────────────────────────────────────
- Separate pages for different features
- Dynamic imports
- Conditional rendering
